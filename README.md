# Universal Converter

Оконное приложение на **Qt (C++)** для точного перевода неотрицательных рациональных чисел из одной позиционной системы счисления в другую.

Поддерживаются основания от 2 до 500, числа с периодическими и конечными дробями, а также цифры, записанные в квадратных скобках (`[222]`). Все вычисления выполняются точно — без `float`/`double` и сторонних библиотек, на собственных классах длинной арифметики.

---

## Возможности

- Перевод **целых чисел**, **конечных дробей** и **периодических дробей**
- Основания систем счисления: **2 ≤ p, q ≤ 500**
- Входная строка длиной до **100 000 символов**
- Автоматическое определение и вывод периода: `0.(3)`, `1.A(B)`
- Цифры в квадратных скобках для значений ≥ 36: `[36]`, `[222][223]`
- Ограничение вывода константой `MAX_OUTPUT_LENGTH = 100000` с маркером `...`
- Валидация входной строки с понятными сообщениями об ошибках

---


## Формат входного числа

Число записывается в системе счисления с основанием **p**.

### Допустимые символы

| Символ | Значение |
|---|---|
| `0`–`9` | Цифры 0–9 |
| `A`–`Z` / `a`–`z` | Цифры 10–35 (регистр не важен) |
| `[N]` | Одна цифра со значением N (для любого N < p) |
| `.` | Разделитель целой и дробной части |
| `(...)` | Период дробной части |

### Примеры корректного ввода

```
123
123.45
0.(3)
1.A(B)
0.12(34)
[222][223]
0.[10][20]([30])
```

### Примеры некорректного ввода

```
.123        — отсутствует целая часть
123.        — отсутствует дробная часть
123.()      — пустой период
123.45(     — не закрыта скобка
```

---

## Формат вывода

Результат записывается в системе счисления с основанием **q** в канонической форме:

- Цифры 0–9 → `0`–`9`
- Цифры 10–35 → `A`–`Z`
- Цифры ≥ 36 → `[N]`

Если дробная часть периодическая — период заключается в скобки. Если конечная — скобки не выводятся.

```
1010
10.25
0.(3)
0.0(0011)
[222][223]
```

Если длина результата превышает `MAX_OUTPUT_LENGTH`, вывод обрезается с добавлением `...`.

## Классы длинной арифметики

### `BigInt`

Неотрицательное целое произвольной длины. Хранение — little-endian вектор «лимбов» по основанию **10⁴**.

| Операция | Сложность | Примечание |
|---|---|---|
| `+`, `-` | O(n) | Классический перенос/заём |
| `* (BigInt)` | O(n^1.585) | **Алгоритм Карацубы** |
| `* (long long)` | O(n) | Скалярное умножение |
| `divmod` | O(n²) | Побитовый поиск по лимбам |
| `gcd` | O(n²) | Алгоритм Евклида через `%` |

#### Алгоритм Карацубы

Умножение двух больших чисел выполняется рекурсивно. При числе лимбов ≤ 32 используется наивный алгоритм O(n²) (база рекурсии). Для больших чисел:

```
a = a1 · BASE^m + a0
b = b1 · BASE^m + b0

z2 = a1 · b1
z0 = a0 · b0
z1 = (a1 + a0) · (b1 + b0) − z2 − z0

a · b = z2 · BASE^(2m) + z1 · BASE^m + z0
```

Вместо 4 рекурсивных умножений — 3, что даёт сложность **O(n^log₂3) ≈ O(n^1.585)**.

### `BigFraction`

Точная рациональная дробь `num/den` в несократимой форме.

#### Построение из цифр (`fromDigits`)

Для числа с предпериодом длины k и периодом длины T в основании p:

```
x = целая_часть + pre/p^k + period / (p^k · (p^T − 1))
```

Общий знаменатель вычисляется точно через `BigInt`, результат сокращается через `gcd`.

#### Перевод в основание q (`toBase`)

Алгоритм длинного деления дробной части:

1. Остаток умножается на q
2. Целая часть от деления на знаменатель — следующая цифра
3. Остатки хранятся в `std::map<string, index>` — как только остаток повторился, найден период

## Парсер входной строки

Разбирает строку символ за символом, поддерживая все форматы цифр (`0–9`, `A–Z`, `[N]`) и структуру числа (целая часть, дробная часть, период). При ошибке выбрасывает `std::runtime_error` с описанием причины, которое отображается в UI.

## Сборка

Требования: **Qt 5.x / Qt 6.x**, компилятор с поддержкой **C++17**.

```bash
git clone https://github.com/vladimirspanov-spec/UniversalConverter.git
cd UniversalConverter
qmake UniversalConverter.pro
make
./UniversalConverter
```

Или через **Qt Creator**: открыть `UniversalConverter.pro` → Build → Run.

## Ограничения

- Отрицательные числа не поддерживаются
- Пробелы внутри числа не допускаются
- Основания ограничены диапазоном 2–500
